# Publication, validation et gestion des versions

## Publication d’un IG
- Utiliser l’IG Publisher pour générer le site web et le package FHIR.
- Publier le site sur un hébergement public (GitHub Pages, serveur institutionnel, etc.).
- Versionner chaque publication (numéro de version, date, changelog).
- Rendre accessibles toutes les versions historiques via une page dédiée.
- Documenter le processus de publication (workflow, scripts, automatisation possible).

## Validation
- Vérifier la conformité FHIR avec l’IG Publisher et les outils de validation HL7/ANS.
- Contrôler la qualité des liens, images, ressources, pages et orthographe.
- Utiliser les rapports QA générés par l’IG Publisher et corriger toutes les erreurs/avertissements avant publication.
- Effectuer des tests de navigation et d’accessibilité sur le site généré.

## Gestion des issues et évolutions
- Mettre à disposition un canal de remontée des problèmes (onglet "Issues" sur GitHub, formulaire dédié, email de contact).
- Documenter chaque issue avec un titre, une description détaillée et, si possible, une proposition de correction.
- Prioriser et planifier les évolutions en fonction des retours utilisateurs et des besoins métiers.
- Communiquer sur les évolutions majeures via changelog ou page d’actualités.

## Historique et archivage
- Maintenir un historique des versions (changelog, page dédiée, tags Git).
- Archiver les anciennes versions et garantir leur accessibilité (liens, page d’archive).
- Documenter les changements entre chaque version (diff, changelog).

Pour plus d’informations, voir la section "Liens utiles" et la FAQ.
